# Methodology — Three-Judge Cross-Vendor LLM Cross-Validation

**What this category answers**: Are the strict A/B/C/D rubric grades reproducible across independent LLM judges drawn from three different model vendors?

**Manuscript reference**: Methods, Supplementary Note 12.

**Folder name**: retained as `methodology_two_judge/` for backward compatibility with the original deposit. The contents now hold the full three-judge cross-validation log on the 12,180-record canonical corpus; the earlier two-judge log is preserved alongside it for provenance.

## Files

- `three_judge_grades.jsonl` — **load-bearing**. 12,180 records each independently graded by all three judges. The `consensus_grade` field on every record is the 2-of-3 majority and is the single grade source for every reported number in the paper.
- `three_judge_kappa.txt` — pairwise Cohen's κ on the three-judge log; agreement breakdown.
- `two_judge_grades.jsonl` — 5,750-record legacy log from the initial two-judge sweep (Opus 4.6 + GPT-4o). Superseded by the three-judge log; kept for provenance.
- `kappa_summary.txt` — short summary file generated by `compute_kappa.py`.

## Three-judge protocol

Each canonical record was independently graded by:

- **Judge 1**: Anthropic Claude Opus 4.6 (Messages API, max_tokens=80, temperature=0)
- **Judge 2**: OpenAI GPT-4o (Chat Completions, max_completion_tokens=80, temperature=0)
- **Judge 3**: Google Gemini-3-Flash-Preview (Gen AI, max_output_tokens=80, temperature=0)

All three judges receive the identical rubric prompt at `prompts/JUDGE_PROMPT.txt`.

**Consensus rule**: 2-of-3 majority across `judge1_grade`, `judge2_grade`, `judge3_grade`. If all three disagree, fall back to Judge 1 (Opus 4.6). The `consensus_grade` field on every record records the result of this rule and is the single grade reported in the paper.

## Headline numbers (three-judge log)

Pairwise Cohen's κ on 12,180 canonical records:

| Pair | κ |
|---|---:|
| Opus 4.6 ↔ GPT-4o | 0.811 |
| Opus 4.6 ↔ Gemini-3-Flash | 0.809 |
| GPT-4o ↔ Gemini-3-Flash | 0.763 |

Agreement breakdown: 86.2% unanimous, 12.5% 2-of-3 majority, 1.3% all-three-differ.

For comparison, the regex `polish_grade` shows much lower agreement with each LLM judge (κ ≈ 0.53 across all three pairs). The regex grader is therefore deposited only as transparency metadata; the consensus from three independent LLM judges is what every reported number rests on.

## Schema (`three_judge_grades.jsonl`)

| Field | Type | Description |
|---|---|---|
| `trial_id` | string | Links back to the canonical record |
| `cell` | string | Cell identifier (or culture/probe for R5/R6) |
| `model` | string | Model identifier |
| `polish_grade` | string \| null | Legacy regex grader output, transparency metadata only (NOT used for any reported number) |
| `judge1_grade` | string | Opus 4.6 grade A/B/C/D |
| `judge1_reason` | string | Opus 4.6 short justification |
| `judge2_grade` | string | GPT-4o grade A/B/C/D |
| `judge2_reason` | string | GPT-4o short justification |
| `judge3_grade` | string | Gemini-3-Flash-Preview grade A/B/C/D |
| `judge3_reason` | string | Gemini-3-Flash-Preview short justification |
| `consensus_grade` | string | 2-of-3 majority; falls back to Judge 1 if all three differ |
| `ts` | float | Unix timestamp |

## Verifying κ

```bash
python ../../scripts/grading/compute_kappa.py
```

This script reads `three_judge_grades.jsonl`, computes pairwise κ across the three judge columns plus the legacy `polish_grade`, and writes the summary to `kappa_summary.txt`.

## Re-running the three-judge sweep

```bash
python ../../scripts/grading/add_three_judge_full.py
```

This adds Judge 3 (Gemini-3-Flash-Preview) and the consensus column to every canonical record and refreshes `three_judge_grades.jsonl`. The dispatch is resumable: records that already carry a `judge3_grade` are skipped.
